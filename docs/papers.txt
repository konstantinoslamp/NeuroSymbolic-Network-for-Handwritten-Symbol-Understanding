Neuro-Symbolic Integration: System Architecture & Build GuideThis document outlines the architecture and training loop for building a hybrid Neuro-Symbolic (NeSy) system capable of simultaneously optimizing neural network weights and evolving non-differentiable symbolic policies from scratch.1. Core Architecture (Extended NeuroLog)The system is built on a compositional architecture that treats the neural and symbolic systems as independent black-box modules. The framework consists of three mandatory components:Neural Module: A standard neural network (e.g., a CNN) that takes raw data and outputs a continuous vector. It must expose three methods: deduce() (forward pass), abduce() (a dummy method for symmetry), and induce() (standard backpropagation).Translator: A predefined function that bridges the modules. It maps the continuous neural output into a discrete set of symbolic inputs/abducibles.Symbolic Module: A logic-based inference engine executing a non-differentiable symbolic policy. The policy follows Machine Coaching (MC) semantics, structuring knowledge as an elaboration-tolerant, prioritized list of rules. It must expose three methods: deduce() (forward reasoning to compute an outcome), abduce() (backward reasoning to find required inputs given a desired outcome), and induce() (to mutate or append new rules to the policy).2. Neural Module Training (Abductive Learning)The neural network learns without requiring the symbolic policy to be differentiable. Instead, it trains via "Semantic Loss" generated through logical abduction:Generate Abductive Feedback: Given a training sample, the system uses the Symbolic Module's abduce() method on the sample's label to find all valid abductive proofs (combinations of inputs that would logically satisfy the target label under the current policy).Compile to SDD: The abductive proofs are compiled into a Sentential Decision Diagram (SDD).Compute Semantic Loss: The system performs Weighted Model Counting (WMC) on the SDD, weighting each atom by the activation values outputted by the neural network. The Semantic Loss is the negative logarithm of this WMC.Backpropagate: This differentiable loss is used to update the Neural Module's weights via standard gradient descent (induce()).3. Symbolic Policy Training (Evolutionary Loop)Because the symbolic policy cannot be trained via gradients, it is learned concurrently with the neural weights using an adaptation of Valiant's Evolvability framework. The system treats each NeSy instance as an "organism".Initialization: Generation 0 starts with a single organism containing an empty symbolic policy and randomly initialized neural weights.Generate Offspring Population: The fittest parent from the previous generation creates a population of offspring by applying combinations of independent neural and symbolic mutations.Symbolic Mutations: Clone the parent's policy ($S_{0}$), add a new random rule ($S_{+}$), or simplify the parent's latest rule by dropping a literal ($S_{\perp}$).Neural Mutations: Inherit the parent's trained neural weights ($N_{pw}$) or re-initialize weights randomly ($N_{rw}$).Train the Population: Every offspring's Neural Module is trained on the dataset for a set number of epochs using the Semantic Loss process outlined in Section 2.Evaluate Relative Fitness: Each offspring is evaluated against a validation set. Relative fitness is calculated by performing a point-by-point comparison of the offspring's predictions (correct, abstain, wrong) against the parent's baseline.Selection: Offspring are categorized into beneficial, neutral, or detrimental groups based on a fitness threshold. The fittest organism is selected to act as the parent for the next generation.Termination: The loop repeats until a maximum number of generations is reached or an early stopping criterion (e.g., $\ge99\%$ accuracy on the validation set) is triggered.4. Required Engineering OptimizationsTo make training populations of NeSy organisms computationally feasible, two specific optimizations are utilized:Operation Caching for WMC: Building SDDs and executing Depth-First Search (DFS) traversals on the CPU for every batch is a massive computational bottleneck. The system must cache the computational graph of the WMC calculation upon its first encounter with a specific SDD structure. Subsequent calculations for the same structure reuse this graph directly on the GPU with the new neural activation values, thereby minimizing costly CPU-GPU synchronization overhead.Parallel Multiprocessing: Because each generation requires training many small neural networks, sequential training is impractically slow. The framework must use multiprocessing to train multiple networks concurrently on a single GPU, ensuring all inter-process objects are serializable.
